Bugly功能和项目设计

功能：
1. 收集java crash、native crash、 anr，并且能监控到卡顿，上报自定义日志。
2. 应用升级、灰度升级、热修复


项目设计:
1. 上报 java crash
2. 上报 javascript 异常
3. 上报 native 异常
4. 上报异常时，添加自定义信息，如用户的手机机型和Android版本信息，客户端的版本信息，用户信息。  -- crash 回调
5. 场景错误，比如一些很重要的业务，如支付，发生的异常，上传场景id到服务器，有利于筛选出重要级别。 -- 设置标签
6. 探索配置了开发设备后，测试设备 和 客户设备之间的差别。
7. 上传自己抛出的异常
8. bugly 日志附加信息。 -- 有限制，最大30k。 可以在发生异常后，有利于恢复异常发生的场景。

应用升级：
1. 应用升级窗口自定义
2. 渠道灰度
3. 网络环境灰度
4. 数量灰度
5. 自定义应用更新窗口

热修复系列：
针对上述所发生的异常，进行在线热修复。
1. java代码修复
2. 资源更换、删除、添加

渠道测试
1. 多渠道打包实现 -- 美团方案 walle
2. 渠道日志上传
